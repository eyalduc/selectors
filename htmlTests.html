Position Modes = This is where the position property comes into play. There are five position values, but these are the four most important (the fifth being inherit). Static – This is the default property and positions itself according to where you’ve placed it in your HTML, ignoring your padding, margins, top and left properties etc. Absolute – The element will be positioned in relation to its first parent element that has a position set other than static. Fixed – The element will be positioned in relation to the browser window. Relative – The best way to describe this would be that the element behaves according to the typical left to right flow of the document. Like letters, words and text in a wordprocessor, relative elements will flow next to each other only breaking onto the next line when there is no more horizontal room in their parent element. This is the setting you want to make sure your divs sit next to each other

What are the display values used for :

*Block means that the element will not tolerate any elements sitting next to it and its height, width, padding, margins etc can be set. In following with our water analogy, we can think of the block mode as being like rough chunks of ice that don’t sit nicely next to each other. *Inline means that an element will tolerate elements sitting next to it and elements like height and width can’t be set. Again, if this we water, inline would be its liquid form, easily flowing back and forth. *Inline-block means the element will tolerate elements sitting next to it while adopting the other attributes of block mode, such as being able to set height and width. Finally, if this were water, think of it as smooth ice cubes that uniformly sit next to each other.
